as the number of processed files increases.

KEY FINDINGS:
-------------

1. MEMORY GROWTH PATTERN:
   - Memory usage grows roughly linearly with the number of files
   - From ~119 MB (50 files) to ~2.8 GB (3000 files)
   - Memory per file starts high (~2.4 MB/file) but stabilizes around 0.9-1.0 MB/file

2. OUTPUT FILE GROWTH:
   - Output JSON file size grows linearly with files processed
   - From ~10 MB (50 files) to ~433 MB (3000 files)
   - Output per file is very consistent: ~0.14-0.15 MB per file
   - This suggests the data structure scales efficiently

3. SCALING BEHAVIOR:
   - Memory per file decreases as more files are processed (more shared words)
   - At 3000 files: ~0.92 MB memory per file, ~0.14 MB output per file
   - Memory-to-output ratio: ~6.4x (memory usage is ~6.4x larger than output file)

4. EFFICIENCY:
   - The consistent output-per-file ratio indicates efficient indexing
   - Word overlap between files helps reduce memory growth per additional file
